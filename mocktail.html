Mocktail - UI/UX Design

Overview

Mocktail is a mobile-first responsive web app for non-alcoholic beverage recipes.
The goal of Mocktail is to provide users the opportunity and avenue to make fun non-alcoholic beverages, instead of an alcoholic option, as they search for healthy alternatives in their lives. They can save these recipes and share them with others on any device. Each element of the interface is to stay true to functionality and simplicity in design for the easiest user experience.

Sample Mockups

The screens below are just a few of the screens found within the app starting with a sample of the various responsive breakpoints and followed by some various mobile screens.

The interface remains simple as the screens grow in size. The navigation remains as a hamburger menu throughout each breakpoint, as seen here in the 'Recipe' screen, in order to keep the focus on the current content the user is viewing.

The 'Log in' screen displays a light, refreshing image to introduce the user to the app as a fun, healthy alternative to other beverage recipe web apps.

As the user enters the app, they are greeted with the 'Home' screen, which uses a two column system to largely display featured recipe options of the moment.

The ability to save mixes and quickly access them was a common need that each potential user mentioned in user interviews. A simple tap or click of the 'Like' icon keeps a particular mix in the user's history.

For personalization, the user can create a profile, which also functions as a hub to visit one's most recent mixes.

A quick tap of the 'Search' icon allows the user to look up a specific drink or use a keyword, such as an ingredient, to learn about some new beverages and access their recipes.

Providing users an assortment of ways to share is paramount and a great way to spread the word about the app.
