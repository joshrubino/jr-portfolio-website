Bored - UI/UX Design

Overview

Bored is an activity generating mobile app designed for both iOS and Android mobile devices using Human Interface and Material Design guidelines respectively.
The goal of this app is to provide an activity or experience to a user they would not have thought to do otherwise. It is a tool that gets people out into the world (using their location or inputting a location) to experience life.

The screens below are just a sampling of a few of the screens found within the app pairing the iOS and Android versions.

After an onboarding process, the user is greeted with the 'Home' screen. It adheres to each deviceâ€™s guidelines, playing off their strengths, such as the use of radio buttons on the Android screen and the Bored button is prominent in each version, attracting the user to generate their activity.

After generating an activity from the 'Home' screen, the user is brought to the 'Activity' screen. The iOS version is scrollable for more info, whereas the Android version uses segmented controls to retrieve further information.

'My Library' uses a gradient into the image on iOS and Android places all the info within a card. Both scroll horizontally for more activities and allow the user to rate their past activities.

Differences pointed out in earlier screens are clear here in the 'Profile' and 'Settings' screens, as well as the iOS version using rounder elements and Android using sharper elements with drop shadows.
